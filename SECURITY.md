## 回報安全漏洞

如果您發現安全漏洞，請**不要**建立公開的 GitHub Issue。

請透過以下方式回報：

1. **Email**: 請發送郵件至專案維護者的電子郵件地址
2. **詳細說明**：
   - 漏洞的詳細描述
   - 重現步驟
   - 可能的影響範圍
   - 建議的修復方案（如果有的話）

我們會在收到回報後：
- 在 48 小時內確認收到回報
- 評估漏洞的嚴重性
- 在修復後發布安全更新
- 在適當的時候公開漏洞資訊（在修復後）

## 安全最佳實踐

使用本 SDK 時，請遵循以下安全最佳實踐：

1. **保護您的憑證**
   - 永遠不要將 `hashKey` 和 `hashIV` 提交到版本控制系統
   - 使用環境變數或安全的配置管理系統
   - 定期輪換您的 API 憑證

2. **使用 HTTPS**
   - 確保所有 API 請求都透過 HTTPS 進行
   - 驗證 SSL/TLS 憑證的有效性

3. **驗證輸入**
   - SDK 已內建輸入驗證，但仍建議在應用層進行額外的驗證
   - 驗證所有使用者輸入

4. **錯誤處理**
   - 不要向使用者暴露敏感的錯誤訊息
   - 記錄錯誤以便追蹤，但不要記錄敏感資訊

5. **保持更新**
   - 定期更新 SDK 到最新版本
   - 關注安全公告和更新日誌

6. **環境隔離**
   - 在測試環境中測試所有變更
   - 使用不同的憑證用於測試和正式環境
